• <img> — Images
    1. Needs a src attribute with a URL, and an alt attribute with descriptive text.
    2. To create an image that is a link, put an img element between the open and close tags of an a element, for instance: <a href="https://google.com"><img src="glogo.png" alt="Google!"></a>
    3. Ex. <img src="https://fakeurl.example.net/fish.png" alt="Alternative Text">
  • <a> — Hyperlinks.
    1. Needs an href attribute with a URL.
    2. a element is an inline element. It doesn't create a box on the page. Links normally have the appearance of a specially highlighted piece of text.
    3. Ex. <a href="https://github.com/">GitHub</a>
  • <ul> & <ol> & <li> — Unordered and ordered lists.
    1. By default, ul lists are displayed with bullet points, and ol lists with numbered items.
    2. The li element has to be nested inside a ul or ol list; it can't occur on its own.
    3. ul for unordered list and ol is for ordered list
    4. The closing tag on the li element is optional. So you can include them or leave them off and it will work the same either way.
    5. Pay attention to the fact that the closing tag is optional for li, but not optional for ol and ul.
  • <em> & <strong> — Emphasis
    1. Tag marks text that has stress emphasis which traditionally means that the text is displayed in italics by the browser.
    2. Any text editor that uses plain text should work. Code editors use plain text and have extra features to make your life easier (such as syntax highlighting). But stay away from word processors for HTML and programming — these add extra data that can break your code!
    3. By default, text inside <em> is displayed as italic, and text in <strong> is displayed as boldface.
  • <p> — Paragraph

    A line break does not create a new paragraph; it only marks the end of a line.

    Text in a paragraph is separated visually from other paragraphs by a small margin.

    The p element is an element for which the closing tag is optional.

    When you leave off an optional closing tag, it is still implied—that is, the browser will figure out where the element should be closed, even though you didn't explicitly tell it.

  • <br> — Line break. (empty)

    A line break does not create a new paragraph; it only marks the end of a line

  • <sub> & <sup> — Subscript and superscript

    sub and sup stand for "subscript" and "superscript". Originally, HTML started out as a tool for scientific and academic work; and these are really useful for science and math, like writing x2+3x+4 = 1 or chemical formulas like H2O.

  • <mark> — Highlighting.

    Not very often used, but it's kind of cool

  • <h1> -> <h6> — Headings

    Elements for headlines or section headings, starting with <h1> for the largest headlines and going down to <h6> for the smallest.

  • div — Division

    Use it to divide up the page into different sections.
